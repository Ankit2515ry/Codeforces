Even Larger</div><div class="time-limit"><div class="property-title">time limit per test</div>2 seconds</div><div class="memory-limit"><div class="property-title">memory limit per test</div>256 megabytes</div><div class="input-file input-standard"><div class="property-title">input</div>standard input</div><div class="output-file output-standard"><div class="property-title">output</div>standard output</div></div><div><p>  </p><p>An array is called <span class="tex-font-style-it">good</span> if, for every subarray$$$^{\text{∗}}$$$ of length at least $$$2$$$, the sum of the elements at even indices (with respect to the <span class="tex-font-style-bf">original array</span>) is greater than or equal to the sum of the elements at odd indices. Array indexing starts from $$$1$$$.</p><p>For example, the arrays $$$[3,8,4,4]$$$ and $$$[2,3,1,4,2]$$$ are good. The array $$$[0,2,4,1]$$$ is not because, in the subarray $$$[2,4,1]$$$, the elements at even indices in the original array are $$$2$$$ (index $$$2$$$) and $$$1$$$ (index $$$4$$$), while the only element at an odd index is $$$4$$$ (index $$$3$$$). Since $$$2 + 1  \lt  4$$$, the condition does <span class="tex-font-style-bf">not</span> hold for this subarray.</p><p>You are given an array of $$$n$$$ <span class="tex-font-style-bf">non-negative</span> integers $$$a_1,a_2,\ldots,a_n$$$. In one operation, you can decrease any element in the array by $$$1$$$, but all elements must remain <span class="tex-font-style-bf">non-negative</span>. Your task is to find the minimum number of operations needed to make the array $$$a$$$ good. It can be proved that it is possible to make the array good using a finite number of operations.</p><div class="statement-footnote"><p>$$$^{\text{∗}}$$$An array $$$b$$$ is a subarray of an array $$$a$$$ if $$$b$$$ can be obtained from $$$a$$$ by the deletion of several (possibly, zero or all) elements from the beginning and several (possibly, zero or all) elements from the end. </p></div></div><div class="input-specification"><div class="section-title">Input</div><p>Each test contains multiple test cases. second line of each test case contains $$$n$$$ non-negative integers $$$a_1,a_2,\ldots,a_n$$$ ($$$0 \le a_i \le 10^9$$$) — the elements of the array $$$a$$$.</p><p>It is guaranteed that the sum of $$$n$$$ over all test cases does not exceed $$$2 \cdot 10^5$$$. </p></div><div class="output-specification"><div class="section-title">Output</div><p>For each test case, output a single integer in a
